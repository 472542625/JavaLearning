引用计数法（已经被淘汰，存在循环依赖问题）

User user = new User();
每个对象有一个年龄，小于或者等于15岁，存放在新生代里面，如果大于15岁，存放在老年代里面
GC线程不定时进行回收时，如果对象被引用，年龄加1，如果没有被回收年龄会减1（如果年龄0岁的话会被垃圾回收机制认为是不可达对象，清理）
（循环依赖问题：a指向b，b指向a，a=null，b=null，都不会被清理）



根搜索算法（GCroots就是被引用对象）
下面对象可以作为GCRoots对象：
1）虚拟机栈中引用的对象
2）方法区中类静态属性引用的对象
3）方法区中常量引用的对象
4）本地方法栈JNI引用的对象

判断是否可达，需要和根节点有依赖关系
如果没有和我的GCroots有任何引用的情况，就认为是不可达对象
